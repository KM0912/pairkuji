# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**ペアくじ (pairkuji)** - ダブルス練習試合の公平な組み合わせ管理PWA

### プロダクトコンセプト
- **公平性が伝わる自動生成**: 最近のペア・対戦履歴を重み付け反映し、結果が直感的に「公平」と感じられる組み合わせを1タップで生成
- **現場スピード最優先**: 片手操作・1タップ生成・Undo/Redo、体育館のフロアで思考停止で運用できるUI
- **オフライン・壊れない**: IndexedDBによる自動保存、PWAキャッシュ、端末クラッシュからの確実な復帰

### プロダクトビジョン
バドミントン・テニス・卓球などダブルス練習の組み合わせ管理において、「公平・効率・納得」を1タップで実現する。手作業くじ引きによる偏り・時間ロス・クレームを解消し、主催者も参加者も練習に集中できる環境を提供する。オフライン完結のPWAとして、通信環境に依存しない安定した運用を実現する。

### 目的
- 試合回数・休憩回数の偏りを最小化し、全参加者が納得できる公平な組み合わせを自動生成する
- ラウンド生成を1秒以内で完了し、現場での待機時間を最小化する
- 途中参加・離脱・体調配慮に即応し、変動に強い練習進行を支援する
- オフライン環境でも完全に動作し、端末クラッシュ後も状態を復元する

## ターゲットユーザー

### ペルソナA: 主催者・進行役（クラブキャプテン / 30代）
- バドミントンサークルの運営責任者。集合〜解散までの進行を担い、限られた時間で効率よく回すことを重視
- スマホ中心（片手操作・ポケット運用）、体育館での利用、通信不安定前提
- **現在の課題**: 手作業くじ引きは偏り・時間ロス・クレームの温床。名簿と紙の二重管理で更新漏れが起きる
- **期待する解決策**: ボタン一つで偏りのない組み合わせが出て、途中参加にも即対応できること
- **典型的なワークフロー**: 参加者チェック → コート数設定 → 組み合わせ生成 → 入替/休憩操作 → 次ラウンド確定 → フルスクリーン掲示

### ペルソナB: 受付・サブ進行（運営サポート / 20代）
- 参加者管理と現場の情報更新を担当。主催者の指示で即時反映する役回り
- タブレット（卓上運用）中心
- **現在の課題**: 紙の名簿と口頭伝達で転記ミス・遅延が発生
- **期待する解決策**: トグル操作で参加/休憩を切り替え、次の生成で自動的に公平に調整されること

### ペルソナC: 一般参加者（プレーヤー / 社会人サークル）
- アプリは基本操作せず、掲示で自分の割り当てを確認する
- **現在の課題**: 同じ相手・ペアの連発や休憩の偏りへの不公平感
- **期待する解決策**: いろいろな人と偏りなく試合でき、待たされすぎないこと

### ペルソナD: 顧問 / コーチ（学校部活動・スクール指導者）
- 練習の質と多様性を重視し、学習効果と公平性の両立を図る
- タブレット中心、指導しながら片手操作
- **現在の課題**: 同カードの連発、途中参加による不公平の発生
- **期待する解決策**: 未対戦・未同ペアの消化を促進し、多様な組み合わせを実現すること

### アンチペルソナ（対象外）
- 公式大会の大規模トーナメント運営（複雑なドロー・審判割り・スコア集計が必要）
- シングルス専用の練習運営（本アプリはダブルス前提）

## 成功指標(KPI)

### プライマリーKPI
- **公平性**: 出場回数の分散（標準偏差）が0.5以下を維持
- **連続休憩率**: 連続休憩2回以上の発生が全体の5%以下
- **生成速度**: ラウンド生成が1秒以内（8〜24名、2〜6面）

### セカンダリーKPI
- **操作効率**: ラウンド生成から掲示までの操作が3タップ以内
- **復元性**: 端末クラッシュ後、直前の状態を100%復元
- **ペア多様性**: 直近5ラウンドでの同一ペア再登場率が10%以下

### 測定方法
- KPIはすべてクライアント側で計算・表示（プライバシー保護のためサーバー送信しない）
- 統計画面で出場/休憩カウント、ペア・対戦の偏りサマリを確認可能

## 機能要件

### コア機能(MVP)

#### F1: メンバー管理

**ユーザーストーリー**:
主催者として、練習参加者の名簿を管理するために、メンバーの追加・編集・有効/無効化機能が欲しい

**受け入れ条件**:
- [ ] メンバーの追加（名前入力のみで登録可能）
- [ ] メンバーの名前編集
- [ ] メンバーのアクティブ/非アクティブ切替
- [ ] メンバーの削除
- [ ] メンバー一覧表示（アクティブ/非アクティブでフィルタ可能）

**優先度**: P0（必須）

#### F2: 練習セッション開始

**ユーザーストーリー**:
主催者として、当日の練習を素早く始めるために、コート数と参加者を選択して練習を開始したい

**受け入れ条件**:
- [ ] コート数の設定（2〜6面）
- [ ] アクティブメンバーから当日の参加者を選択
- [ ] 参加者は最低4名以上（1コート分）
- [ ] 前回の練習データがある場合は継続/リセットを選択可能

**優先度**: P0（必須）

#### F3: 公平な組み合わせ自動生成

**ユーザーストーリー**:
主催者として、全員が納得できる組み合わせを素早く作るために、公平性アルゴリズムによる自動生成が欲しい

**受け入れ条件**:
- [ ] 1タップでラウンドを生成
- [ ] 8〜24名、2〜6面で1秒以内に生成完了
- [ ] 試合回数・休憩回数の偏りを最小化
- [ ] 直近のペア・対戦の重複を減衰付きペナルティで回避
- [ ] 連続休憩2回以上の発生率を5%以下に抑制
- [ ] 途中参加者に対する試合回数補正（playedOffset）

**優先度**: P0（必須）

#### F4: ラウンド進行操作

**ユーザーストーリー**:
主催者として、試合の進行を柔軟に管理するために、コートごとの入替・休憩操作ができたい

**受け入れ条件**:
- [ ] コートカードUIでペア・対戦相手を一覧表示
- [ ] 休憩者一覧の表示
- [ ] 参加者のステータス切替（active/rest）
- [ ] コート数の変更（練習中でも可能）
- [ ] ラウンドの再生成
- [ ] 途中参加者の追加

**優先度**: P0（必須）

#### F5: フルスクリーン掲示

**ユーザーストーリー**:
一般参加者として、自分のコート割り当てを素早く確認するために、見やすいフルスクリーン表示が欲しい

**受け入れ条件**:
- [ ] コートごとのブロック表示（大きな文字）
- [ ] 休憩者の区別表示
- [ ] 体育館の照明下でも読みやすいコントラスト
- [ ] コート識別色（最大6色）と番号の併用（色覚多様性配慮）

**優先度**: P0（必須）

#### F6: データ永続化・復元

**ユーザーストーリー**:
主催者として、端末トラブルがあっても練習を継続するために、自動保存と状態復元が欲しい

**受け入れ条件**:
- [ ] すべての状態変更をIndexedDBに自動保存
- [ ] アプリ再起動時に直前の練習状態を復元
- [ ] 各レコードにupdatedAtタイムスタンプを保持

**優先度**: P0（必須）

#### F7: PWA対応

**ユーザーストーリー**:
主催者として、体育館のオフライン環境でも使えるように、PWAとしてインストール・動作してほしい

**受け入れ条件**:
- [ ] Service Workerによるオフライン動作
- [ ] ホーム画面へのインストール（マニフェスト設定）
- [ ] 静的アセットのCache First戦略
- [ ] オフライン起動の確実性

**優先度**: P0（必須）

#### F8: 統計表示

**ユーザーストーリー**:
顧問・コーチとして、練習の公平性を確認するために、出場・休憩回数やペア・対戦の偏りを一覧で見たい

**受け入れ条件**:
- [ ] 各参加者の出場回数・休憩回数の表示
- [ ] ペア頻度統計の表示
- [ ] 対戦頻度統計の表示
- [ ] ラウンド履歴の表示

**優先度**: P0（必須）

### 将来的な機能(Post-MVP)

#### F9: 偏りサマリと改善ヒント

分散・レンジ・連続休憩率の可視化と、アルゴリズムの重み調整UIを提供する。

**優先度**: P1（重要）

#### F10: エクスポート機能

練習結果のCSV/画像エクスポートと簡易バックアップ機能。

**優先度**: P1（重要）

#### F11: キオスク表示最適化

自動画面ロック防止、明暗テーマ切替、掲示専用モードの強化。

**優先度**: P1（重要）

#### F12: ルールプリセット

部活・社会人・初心者配慮など、場面に応じたアルゴリズム重み付けのプリセット。

**優先度**: P2（できれば）

#### F13: クラウド同期

任意のクラウド同期機能。オフライン単独動作を維持しつつ、複数端末でのデータ共有を可能にする。

**優先度**: P2（できれば）

## 非機能要件

### パフォーマンス
- ラウンド生成: 8〜24名、2〜6面で1秒以内
- 初回ロード: PWAキャッシュ後1秒未満
- UI応答: すべてのユーザー操作に100ms以内で反応

### ユーザビリティ
- 新規ユーザーが初回操作で練習開始まで到達できる（複雑な設定不要）
- 片手操作（スマートフォン）で全機能を操作可能
- 体育館の強い照明下で視認可能な文字サイズ・コントラスト（本文コントラスト比7.0:1以上）
- 大きなタップ領域の確保

### 信頼性
- すべての状態変更でupdatedAtタイムスタンプを更新し、クラッシュ後も復元可能
- データ損失ゼロ（自動保存による）

### セキュリティ・プライバシー
- データはローカル保存を基本（サーバー送信なし）
- 保存する個人情報は氏名のみ（追跡・分析なし）
- セキュリティヘッダー設定（X-Frame-Options、X-Content-Type-Options等）

### アクセシビリティ
- 文字コントラスト: 本文・ラベル 7.0:1以上、ボタンテキスト 4.5:1以上
- 非文字UI（アイコン・境界）: 3.0:1以上
- 色覚多様性対応: 色と番号・アイコン・パターンの併用（色のみで識別しない）
- キーボード操作配慮

### 対応環境
- 参加者: 8〜24名
- コート: 2〜6面
- 端末: スマートフォン（iOS/Android）、タブレット
- ブラウザ: Chrome、Safari（最新2バージョン）
- ネットワーク: オフライン完全対応

## スコープ外

明示的にスコープ外とする項目:
- 公式大会のトーナメント管理（ドロー・審判割り・スコア集計）
- シングルスの組み合わせ管理
- ユーザーアカウント・認証機能（MVPでは不要）
- サーバーサイドのデータ保存・分析
- 多言語対応（日本語のみ）
- スコア記録（詳細なポイント管理。勝敗記録はF9として実装済み）
